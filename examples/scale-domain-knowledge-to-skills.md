# Example: Scale-stage team converting knowledge into Skills

## User prompt

```text
Use AI-Native Founder Orchestrator.

We have repeatable demand.
Product: compliance automation for healthcare clinics.
Team: 7 people.
Problem: customer onboarding depends on two domain experts. They answer the same questions, review the same documents, and correct the same mistakes every week.

Convert company knowledge into Skills.
```

## Expected output style

```text
Stage: Scale

Why this stage:
The company has repeatable demand and repeated operating knowledge. The bottleneck is not only sales or product; it is domain knowledge trapped in experts.

Founder-only decisions:
- Decide which compliance judgments must remain human-reviewed.
- Decide what level of automation is acceptable for regulated customer work.
- Decide which knowledge creates advantage and should become internal infrastructure.

Claude Chat tasks:
- Convert repeated expert decisions into explicit rules and review questions.
- Draft onboarding playbooks and exception handling policies.
- Red-team automation boundaries.

Claude Code tasks:
- Create internal checklists, validation scripts, document review tooling, and eval fixtures.
- Build repo-level Skills only after the rules are stable.

Claude Cowork tasks:
- Analyze onboarding docs, customer Q&A, review notes, policy documents, and expert corrections.
- Extract patterns, recurring edge cases, and decision rules.

Skills / memory / workflow opportunities:
- Clinic onboarding document review Skill.
- Compliance exception triage Skill.
- Customer implementation checklist memory.
- Sales objection knowledge Skill.

This week's artifact:
Domain knowledge to Skill map.

Exit criteria:
- A new teammate or AI agent can handle 60-70% of routine onboarding review with expert oversight.
- Exceptions are clearly routed to humans.
- Repeated expert corrections become documented decision rules.

AI-native failure mode:
Failure: company knowledge stays trapped in people and scattered docs while headcount grows around the bottleneck.
Countermeasure this week: convert one repeated expert review into a Skill spec with trigger, inputs, workflow, output, and validation.
```
